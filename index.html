Carian & Alat Koordinat

📍 Cari Koordinat (Lat, Long)
🏠 1. Cari Tempat (Jalan / Masjid / Sekolah / dll)
Carian dikuasakan oleh OpenStreetMap Nominatim (data awam) — dihadkan pada penggunaan wajar.
📜 2. Cari No. Lot / Mukim
Butang MyLOT JUPEM, iPlan, dan k-GIS KESEDAR menghubungi pelayan ArcGIS/geocode rasmi masing-masing (gismaps.jupem.gov.my, iplan.planmalaysia.gov.my, k-gis.kesedar.gov.my). Jika sekatan CORS wujud, sistem cuba proxy awam & menyediakan pautan buka portal terus. Butang Lapisan Diimport menyemak atribut SHP tempatan (medan LOT / MUKIM).
🧭 3. Penukar Koordinat (RSO / Cassini)
Parameter unjuran berdasarkan definisi rasmi EPSG / JUPEM (RSO Malaya (Semenanjung): EPSG:3168 (Datum Kertau 1948), grid Cassini negeri: EPSG:3377–3385 & 4390–4398). WGS84 dianggap setara GDM2000 (beza <1m). Untuk kerja ukur / kadaster rasmi, sahkan semula dengan JUPEM.
📥 Banyak Koordinat / Import Fail
Setiap baris: koordinat1, koordinat2, label(pilihan) — dipisah koma, tab atau ruang. Fail CSV/TXT diimport terus ke kotak di atas untuk semakan sebelum ditukar. Arah & sistem sasaran ikut tetapan di atas.
📐 4. Import / Eksport DWG · DXF · KML · KMZ
Pelayar web tidak boleh baca/tulis fail .dwg binari terus (format tertutup Autodesk). Alat ini guna .dxf — format pertukaran CAD terbuka yang serasi penuh dengan AutoCAD/Civil3D. Buka fail .dxf dalam AutoCAD dan "Save As → DWG" untuk tukar terus ke .dwg tanpa kehilangan data.
Bentuk asal dikekalkan sepenuhnya: titik (POINT) kekal titik; garisan/polyline (LINE, LWPOLYLINE/POLYLINE, KML LineString) kekal bersambung sebagai satu garisan — bukan bertaburan jadi titik berasingan; polyline tertutup & Polygon jadi kawasan tertutup. Objek terus ditambah sebagai lapisan baharu pada tab "Peta" mengikut Sistem Koordinat Dalam Fail di atas, dan senarai koordinatnya turut diisi ke jadual "Banyak Koordinat" untuk rujukan/eksport. Guna ini apabila fail sebenarnya menyimpan koordinat grid tempatan (Cassini/RSO) dan bukan Lat/Long WGS84 piawai.
Guna hasil semasa dalam jadual "Banyak Koordinat" di atas (lepas klik Tukar & Papar Semua, atau lepas Import DXF/KML/KMZ) sebagai sumber titik.
Format DWG/DXF: koordinat ditulis terus mengikut sistem sasaran (unit meter, atau darjah jika WGS84) — serasi AutoCAD/Civil3D.
Pilih jenis objek — hanya jenis yang dipilih akan ditulis ke fail eksport (DXF/KML/KMZ). Label = nama placemark; Imej = pautan <description> ke gambar jika tersedia dalam ciri.
🌐 5. QuickOSM — Muat Data OpenStreetMap
Setara fungsi QuickOSM di QGIS: tanya data OpenStreetMap terus melalui Overpass API dan tambah sebagai lapisan baharu pada peta. Semua ciri (titik / garisan / poligon) yang dimuat boleh dieksport ke DXF / KML / KMZ menggunakan panel di atas.

🧱 Tapak Kerja Tanah (Cut & Fill / Pokok)

Guna alat 🧱 Tanda Tapak pada bar alat di atas peta untuk lukis poligon tapak. Poligon yang sama dikongsi untuk pengiraan isipadu gali (cut), isipadu tambak (fill) dan anggaran bilangan pokok di bawah.
Belum ada tapak ditanda. Klik butang 🧱 di atas.

⛏️ Isipadu Tanah Gali (Cut)

Isipadu = Luas tapak × Kedalaman gali, ditambah peruntukan cerun galian (batter) dan peratus runtuhan/pelonggaran tanah (soil bulking).

🚧 Isipadu Tanah Tambak (Fill)

Isipadu = Luas tapak × Ketinggian tambak, ditambah peruntukan cerun tambakan (batter) dan peratus allowance ampaian/pemampatan (settlement/collapse) tanah tambak.

🌳 Anggaran Bilangan Pokok

Anggaran bilangan pokok berdasarkan jarak tanaman grid segiempat sama di dalam kawasan tapak, ditolak jalur tepi/buffer dari sempadan. Nilai indikatif untuk perancangan awal.
